1. Discuss the scope of var, let, and const

Var...

Block Scope var’s are not block-scoped: scope is not limited to the block it was defined in. i was still in the “global scope” within the if block. i's value was overwritten, which may have not been the intention. Usually We are not use var proparlly We are Only Use let an const

Let...

let variables are block-scoped! Specific scope = less mistakes. Even though i was assigned to 1 in the if block, that assignment was local to the block and therefore our “global” i was still 0. The if block’s scope was separate from the global scope.

Const...

const restricts over-writing variables. const i = 0;i = 1; // TypeError: Assignment to constant variable. const doesn’t even let you declare a variable without assigning its (constant) value! const i; // SyntaxError: Missing initializer in const declaration const, like let, is block scoped. const does allow variable mutation (only objects/arrays are mutable in JS).

2. Tell us the use cases of null and undefined

Undefined...

undefined is a global variable that JavaScript creates at run time. JavaScript assigns undefined to any variable that has been declared but not initialized or defined. In other words, in a case where no value has been explicitly assigned to the variable, JavaScript calls it undefined.

Null...

It is one of JavaScript’s primitive values and is treated as falsy for boolean operations.null is an empty or non-existent value and null must be assigned. We use null when we want to explicitly declare that a variable is empty.
